## 1. 一句話定位

**App 名：AI老友記**（已鎖定；副題：老友記嘅 AI 生活學堂）。
教香港「活躍初老」（60–72 歲）長者，由淺入深學識將 AI 融入日常生活嘅教學 App。
內容定時更新，全程大字、撳掣操作、唔使打字。

## 2. 目標用家（已鎖定）

- **主 persona**：活躍初老 60–72 歲。識用 WhatsApp / YouTube / 網上銀行，有智能手機基礎，但未掂過 AI。有自學動機（唔想被時代拋離）。
- **次要角色**：仔女 — 透過 optional「家人同行」功能睇到父母學習進度、俾鼓勵。**唔強制綁定**，長者可隨時閂分享。
- 語言：繁體中文＋廣東話口語文案（介面文案要似同長者傾偈，唔好書面語）。

## 3. 商業模式（已鎖定方向，細節未決）

- 兩條路都保留：**A. 收費**（訂閱 / 仔女買俾父母）同 **C. 社會項目**（NGO / 數碼共融基金合作）。
- MVP 階段：**全免費**，唔起 payment。Code 層面預留 feature flag 方便日後鎖 premium 內容。
- ❓未決：定價、基金申請對象 — 由 Stephanie 之後決定，MVP 唔使理。

## 4. 出街形式（已鎖定：先 B 後 A）

1. **Phase 1（而家做）：PWA** — 一條 link 用到，加到主畫面，離線可睇已下載課程。
2. Phase 2（未來）：內容同用戶跑通後先包原生 App 上架。MVP 唔使做。

## 5. 課程設計（已鎖定）

### 5.1 三層由淺入深

| 層 | 名 | 內容例子 |
|---|---|---|
| 第一層 | AI 入門（淺） | AI 係咩、點同 AI 講嘢、語音輸入、影相問 AI |
| 第二層 | 生活應用（中） | 問藥物資料、寫祝壽詞、睇唔明嘅信影相翻譯、搵食譜、同 AI 傾偈解悶 |
| 第三層 | 進階玩法（深） | AI 執靚張相、計劃旅行、幫手覆 WhatsApp、AI 語音助手日常化 |

- 完成上一層先解鎖下一層（進度感 + 唔會一開波就嚇親）。
- **防騙必修班**獨立成章、任何時候都入到：AI 假電話、deepfake 假影片、假訊息點認。呢部分內容要人手把關最嚴。

### 5.2 課堂結構（每課固定三步，參考 mockup）

1. **點解要學** — 一段口語解釋 + 生活痛點
2. **睇示範** — 模擬對話 demo（user bubble + AI bubble）
3. **考一考** — 1 條二選一大掣選擇題，答啱先完成，答錯有溫柔提示再試

每步都有「🔊 讀出嚟俾我聽」掣（TTS，廣東話優先）。

### 5.3 課程類型

- 場景課（主軸）＋工具課（教 ChatGPT / Gemini / 手機 AI 功能操作，內容要易更新因為工具 UI 成日改）＋防騙課（必修）。

## 6. 內容管道（2026-07-19 更新：AI 生成即自動出街，唔再係人手 approve gate）

- 課程內容以**結構化檔案**儲存（JSON 或 MD + frontmatter），一課一檔，包含：層級、場景、三步內容、quiz、TTS 文本。
- **更新流程**：AI 生成新課 → 直接 `status='published'` 出街，唔使等 approve。
- 節奏：**每週 2–3 課新內容**。
- **2026-07-19 政策改變（Stephanie 拍板）**：原本「approve flow 必須存在，嚴禁全自動出街」呢條規已取消，佢信得過 DeepSeek 嘅生成質素，唔想再逐課人手撳approve。Admin 介面（`admin/app.py`）保留，但改做事後補救用（睇/edit/unpublish 已出街嘅課），唔再係出街前必經嘅gate。
- 生成排程用邊個機制（GitHub Action / cron / Cowork scheduled task）由 Claude Code 建議，揀最慳功夫嗰個。

## 7. UI 鐵律（已鎖定，唔可以妥協）

1. **最細字 22px**，標題 30px+，高對比配色（參考 mockup：米白底 #faf8f4 + 深綠 #2f6f4f 主色）。
2. **全程撳掣，唔使打字**。任何 input 都用大掣選項代替鍵盤。
3. 觸控目標最少 60px 高，掣同掣之間有足夠空位（防誤觸）。
4. 每頁一個主要動作，唔好塞太多嘢。
5. 底部四個大 tab：主頁 / 上堂 / 進度 / 家人（見 mockup）。
6. 語氣：廣東話、鼓勵性、當用家係精明大人而唔係細路（「好叻呀」可以，居高臨下唔得）。
7. 進度感：streak（連續學習日數）、獎章、層級進度條 — 全部已喺 mockup 示範。
8. 「唔識就撳我」求助入口喺主頁常駐。

## 8. 家人同行（optional feature）

- 長者一方：開關掣控制分享；可以撳「👍 覆返個讚」回應家人留言。
- 仔女一方：透過邀請 link（WhatsApp 分享）加入，睇到進度 + 可以留言鼓勵。
- 邀請流程要簡單到仔女幫長者 set 一次就得。

## 9. 技術建議（Claude Code 可自行裁量，但要符合以下原則）

- PWA：任何主流輕量 stack 都得（建議 Vite + React 或類似），single deploy（Vercel）。
- 後端／資料：建議 Supabase（Stephanie 已有帳戶）— 用戶進度、家人關係、課程內容、pending 隊列。
- 登入：**唔可以要長者打字**。建議：仔女/工作坊幫手 set 一次（電話號碼 OTP 由幫手代操作），之後裝置長期保持登入。
- TTS：優先廣東話。Web Speech API 得就用；唔得就預先生成音檔（課程 approve 時一併生成）。
- 離線：已解鎖課程 cache 落本機。
- Analytics：最基本 — 邊課有人開始但冇完成（即係太難／太悶），俾 Stephanie 睇。

## 10. MVP 範圍界線

**要做**：PWA 四個 tab、三層課程引擎、防騙章節、TTS、streak/獎章、家人同行、admin approve 介面、AI 生成課程 pipeline、首批 8–10 課種子內容（AI 生成 + 標明待 Stephanie approve）。

**唔做（Phase 2+）**：原生 app 上架、付費、多語言、社群功能、真人客服。
